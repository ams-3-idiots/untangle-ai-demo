# AGENTS.md

이 문서는 untangle-ai 저장소에서 작업하는 모든 AI 에이전트가 따라야 할 지침이다.

## 작업 시작 전 필수 절차

**어떤 작업(코드 작성·수정·리뷰·설계·디버깅 등)이든 시작하기 전에, 반드시 `docs/` 폴더의 문서를 먼저 읽고 프로젝트 맥락을 파악한다.** 이 단계는 생략할 수 없으며, 요청이 사소해 보이더라도 동일하게 적용한다.

| 문서 | 내용 | 언제 읽나 |
| --- | --- | --- |
| `docs/PROJECT.md` | 프로젝트 목적·핵심 사용자·핵심 가치·주요 AI 기능 개요 | 프로젝트 목표 확인 |
| `docs/FEATURES.md` | 기능별 상세 요구사항(F-번호)·인수 기준(AC)·우선순위(P0/P1) | 구현 범위 확인 |
| `docs/ENVIRONMENT.md` | 기술 스택: uv, python, streamlit, openai, claude | 환경·의존성·실행 관련 작업 시 |
| `docs/ARCHITECTURE.md` | 시스템 구조·모듈 설계 (현재 비어 있음, 작성 예정) | 구조 변경·새 모듈 추가 시 |

## 문서를 읽은 뒤 지켜야 할 핵심 원칙

- **GP-1 (사용자 확정 우선):** 모든 AI 출력은 사용자가 확정하기 전까지 실제 데이터에 반영하지 않는다. 자동 결정 금지.
- **우선순위 준수:** 요구사항에는 P0(필수)·P1(개선) 우선순위가 있다. 구현·제안 시 P0를 먼저 충족한다.
- **인수 기준 충족:** 특정 기능을 건드릴 때는 해당 기능의 인수 기준(AC)을 만족하는지 확인한다.
- **사용자에게 결정권:** AI는 선택지를 제시할 뿐, 최종 결정은 사용자가 한다.

## 작업 방식

- 요구사항과 충돌하거나 문서에 없는 결정이 필요하면, 임의로 진행하지 말고 먼저 사용자에게 확인한다.
- 문서 내용과 실제 코드가 다를 경우, 추측으로 메우지 말고 그 차이를 사용자에게 알린다.
- 작업 후에는 변경 사항이 관련 요구사항(F-번호)·인수 기준(AC)을 충족하는지 스스로 점검한다.
